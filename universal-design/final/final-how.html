How to design for everyone

After understanding why accessibility is important and what it is, it is now time to learn how to implement what you learned. HTML (HyperText markup Language) is the standard language for the web. Accompanied by CSS (cascading style sheets), HTML can make beautiful and accessible websites. Learning how to code accessible from the start will set you up for success. The good news is that the newest version of HTML5 already comes with assistive technology that will help you integrate accessibility without any extra work. Where HTML5 falls short, there is technology called ARIA that helps make accessibility more accessible to coders. ARIA (Accessible Rich Internet Applications) are a set of attributes that can be used in combination with HTML elements that tell the web how to make elements accessible. Specifically, ARIA can help enhance the use of screen readers and keyboard users. ARIA elements can help give descriptions and context around elements on the page, like images.

The best way to ensure your website is accessible is to follow WCAG (Web Content Accessibility Guidelines). These are guidelines that were made to set standards for accessibility in technology. They follow four main principles: perceivable, operable, understandable and robust. WCAG has 13 guidelines that fall under the four principles and they each have testable success criteria.

Coding

If you are learning how to code, use websites like MDN Web Docs to learn how to add accessibility to certain elements. Start small and start simple. Once you have made your webpage be sure to test it against standards to ensure it is accessible.

Important things to keep in mind are alternative texts for images, information architecture like headings, interactions and overall visual cues. A useful resource is this checklist by WEB AIM to ensure your website is ready for publication.

It can also be helpful to keep in mind the 10 usability heuristics to access a website:

  • Visibility of system status
  • Match between system and real world
  • User control and freedom
  • Consistency and standards
  • Error prevention
  • Recognition rather than recall
  • Flexibility and efficiency of use
  • Aesthetic and minimalist design
  • Help users recognize, diagnose, and recover from errors
  • Help and documentation

There are also very powerful web accessibility tools that can check your website automatically and can be downloaded as a web extension. Web AIM’s wave evaluation tool that will point out issues like contrast issues and structural issues. It is recommended to use these resources as you use and update your webpage.
